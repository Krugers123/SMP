# Governance

SMP is intended as an open protocol for human-centered AI communication.

## Governance Principles

- Keep the protocol inspectable.
- Keep human authority explicit.
- Prefer simple public structures over hidden complexity.
- Separate public protocol design from private implementation details.
- Avoid claims that SMP guarantees alignment, correctness, or safety.
- Treat high-impact use cases with additional review and domain-specific governance.

## Versioning

SMP uses draft protocol versions.

The current draft is:

```text
SMP v0.1
```

Breaking changes should be documented before adoption.

## Contribution Direction

Early contributions should focus on:

- clearer packet fields,
- better neutral examples,
- validators,
- interoperability notes,
- human factors research,
- accessibility of structured interaction.
